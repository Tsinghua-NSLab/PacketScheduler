Detailed Description

The simulator is responsible for offering general services to all other interacting parts, such as configurations and current time management. It uses a deterministic priority queue to precisely execute event after event.

Create a random number generator which guarantees the same sequence when the same universal seed is fed in setup().

This is used when a section wants to be independent of whatever configuration has come before it that may have used the universal random number generator. An example is the flow generation.
