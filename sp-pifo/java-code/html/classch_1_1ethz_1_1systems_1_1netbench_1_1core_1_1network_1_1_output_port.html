Detailed Description

Abstraction for an output port on a network device.

There is no corresponding InputPort class, as the output port already forces a rate limit. OutputPort's subclasses are forced to handle the enqueuing of packets, and are allowed to drop packets depending on their own drop strategy to handle congestion at the port (e.g. tail-drop, RED, ...).
